# 2.2.5. 结论

本节应该已经说明，访问 DRAM 并不是一个可以任意加快的过程。至少，与处理器运行速度以及访问寄存器和缓存的速度相比，它并不快。务必记住 CPU 与内存频率之间的差异。一颗运行在 2.933GHz 的 Intel Core 2 处理器与一条 1.066GHz FSB 的时钟比（clock ratio）为 11:1（注意：1.066GHz 总线是四倍泵送的）。内存总线每停顿一个周期，就意味着处理器停顿 11 个周期。对大多数机器来说，实际使用的 DRAM 更慢，因此还会进一步增加延迟。在后续章节讨论停顿时，请记住这些数字。

读取命令的时序图显示，DRAM 模块能够达到很高的持续数据传输速率（sustained data rate）。整个 DRAM 行可以毫无停顿地传输。数据总线可以保持 100% 占用。对 DDR 模块来说，这意味着每个周期传输两个 64 bit 数据字。使用 DDR2-800 模块和双通道时，这代表 12.8GB/s 的速率。

但是，除非专门这样设计，DRAM 访问并不总是顺序的。使用不连续的内存区域，意味着需要预充电以及新的 $\overline{\text{RAS}}$ 信号。这正是速度慢下来、DRAM 模块需要帮助的时候。越早进行预充电并发送 $\overline{\text{RAS}}$ 信号，真正使用该行时的损失（penalty）就越小。

硬件和软件预取（prefetch，见 6.3 节）可以用于在时序上创造更多重叠，并减少停顿。预取也有助于在时间上移动内存操作，使之后真正需要数据之前的争用减少。当一轮中产生的数据必须被存储，而下一轮所需的数据又必须被读取时，这就是一个常见问题。通过在时间上移动读取操作，写入和读取操作就不必在基本相同的时间发出。
