# 2.2. DRAM 访问技术细节

在介绍 DRAM 的章节中，我们看到 DRAM 芯片会对地址进行多路复用，以节省地址引脚这类资源。我们也看到，访问 DRAM 存储单元需要时间，因为这些单元中的电容无法瞬时放电并产生稳定信号；同时，DRAM 存储单元还必须被刷新。现在，是时候把这些因素放在一起，看看它们如何决定 DRAM 访问必须怎样进行。

我们会聚焦当前技术；不会讨论异步（asynchronous）DRAM 及其变体，因为它们已经不再相关。对这个主题感兴趣的读者可参见 [3] 与 [19]。我们也不会讨论 Rambus DRAM（RDRAM），即使这项技术并未过时；它只是没有被广泛用于系统内存。我们只会聚焦同步 DRAM（Synchronous DRAM，SDRAM）及其后继者双倍数据速率 DRAM（Double Data Rate DRAM，DDR）。

**同步 DRAM 顾名思义，其工作方式相对于某个时间源（time source）而言。内存控制器提供一个时钟（clock），该时钟频率决定了前端总线（FSB）**，也就是 DRAM 芯片所用内存控制器接口的速度。在撰写本文时，可用频率包括 800MHz、1,066MHz，甚至 1,333MHz，并且下一代还宣称会达到更高频率（1,600MHz）。这并不意味着总线实际使用的频率真的这么高。相反，如今的总线采用二倍或四倍泵送（double- or quad-pumped），也就是说每个周期传输两次或四次数据。更高数字更好卖，所以厂商喜欢把四倍泵送的 200MHz 总线宣传成“有效（effective）”800MHz 总线。

对于如今的 SDRAM 来说，每次数据传输由 64 bit（即 8 byte）组成。因此，FSB 的传输率等于 8 byte 乘以有效总线频率（对四倍泵送的 200MHz 总线来说是 6.4GB/s）。这听起来很多，但它只是突发速度，也就是永远不会被超过的最大速度。正如我们接下来会看到的，与 RAM 模块通信的协议存在大量停顿时间（downtime），在这些时间里无法传输数据。我们必须理解并尽量减少的，正是这些停顿时间，只有这样才能达到最佳性能。
