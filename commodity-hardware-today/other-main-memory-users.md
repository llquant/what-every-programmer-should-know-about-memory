# 2.3. 其他主内存用户

除了 CPU 之外，还有其他系统组件可以访问主内存。网络控制器、大容量存储控制器等高性能扩展卡（card），无法承受把所有需要或提供的数据都通过 CPU 传输的成本。相反，它们会直接从主内存读取数据，或直接向主内存写入数据（直接内存访问〔Direct Memory Access〕，DMA）。在图 2.1 中可以看到，扩展卡可以通过南桥和北桥直接与内存通信。其他总线，例如 USB，也需要 FSB 带宽，即使它们不使用 DMA 也是如此，因为南桥同样会经由北桥，通过 FSB 连接到处理器。

虽然 DMA 无疑有益，但它意味着 FSB 带宽上会出现更多竞争。在 DMA 流量很高时，CPU 等待来自主内存的数据时，可能比平常停顿更久。如果有合适的硬件，也有办法绕过这个问题。采用图 2.3 那样的架构时，可以设法确保计算使用不受 DMA 影响的节点上的内存。也可以为每个节点连接一个南桥，从而把每个节点上 FSB 的负载均匀分散。可能性有很多。第六节将介绍一些技术和编程接口，它们有助于通过软件实现可能的改进。

最后还应该提到，有些廉价系统的图形系统没有独立、专用的视频 RAM（video RAM）。这些系统会使用一部分主内存作为视频 RAM。由于视频 RAM 访问很频繁（对于 1024x768、16 bpp、60Hz 的显示器来说，就是 94MB/s），而且系统内存不像显卡上的 RAM 那样有两个端口，这会显著影响系统性能，尤其是延迟。如果性能是优先事项，最好避开这类系统。它们带来的麻烦多于价值。购买这些机器的人应该知道，自己无法获得最佳性能。
