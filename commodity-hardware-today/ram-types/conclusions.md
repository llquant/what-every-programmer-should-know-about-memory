# 2.1.4. 结论

如果本节中的细节有点令人招架不住，不必担心。需要从本节带走的重点是：

* 并非所有内存都使用 SRAM 是有原因的。
* 存储单元需要被逐个选择后才能使用。
* 地址线数量直接影响内存控制器、主板（motherboard）、DRAM 模块和 DRAM 芯片的成本。
* 读取或写入操作的结果可用之前，需要经过一段时间。

下一节会更详细地讨论实际访问 DRAM 内存的过程。我们不会更深入讨论 SRAM 的访问方式，因为 SRAM 通常直接寻址。这样做是出于速度考虑，也因为 SRAM 内存规模有限。SRAM 目前用于 CPU 缓存和片上（on-die）存储，在这些位置连接较少，并且完全受 CPU 设计者控制。CPU 缓存是后文会讨论的主题；这里我们只需要知道，SRAM 存储单元存在某个最大速度，这取决于在 SRAM 上投入的设计成本。这个速度可能只是略慢于 CPU 核心，也可能慢一到两个数量级。
