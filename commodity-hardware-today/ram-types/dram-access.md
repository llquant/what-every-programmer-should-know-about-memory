# 2.1.3. DRAM 访问

程序使用虚拟地址（virtual address）来选择内存位置。处理器将其转换（translate）为物理地址（physical address），最后由内存控制器选择与该地址对应的 RAM 芯片。为了选择 RAM 芯片中的单个存储单元，物理地址的一部分会以若干地址线（address line）的形式传递进去。

让内存控制器逐个寻址（address）内存位置完全不现实：4GB RAM 会需要 2<sup>32</sup> 条地址线。作为替代，地址会使用一组更少的地址线，以二进制数值编码后传递。以这种方式传给 DRAM 芯片的地址必须先被解复用（demultiplex）。一个有 $N$ 条地址线的解复用器（demultiplexer）会有 $2^N$ 条输出线（output line）。这些输出线可以用于选择存储单元。对容量较小的芯片来说，这种直接方法没有太大问题。

但如果存储单元数量增加，这种方法就不再合适。一个容量为 1Gbit[^6] 的芯片需要 30 条地址线和 2<sup>30</sup> 条选择线（select line）。在不牺牲速度的前提下，解复用器的尺寸会随输入线（input line）数量呈指数增长。用于 30 条地址线的解复用器需要大量芯片面积，还会带来解复用器本身在尺寸和时间上的复杂度。更重要的是，在地址线上同步传输 30 个脉冲（impulse），比“只”传输 15 个脉冲困难得多。需要按完全相同长度布线或适当安排时序的线路越少，设计就越容易。[^7]

图 2.7 以非常高层的方式展示了 DRAM 芯片。DRAM 存储单元被组织为行（row）和列（column）。它们也可以全都排在一行里，但那样 DRAM 芯片就需要一个巨大的解复用器。采用数组（array）方式后，这个设计只需要一个解复用器和一个多路复用器，而且两者尺寸都只有原来的一半。[^8]这在各方面都是巨大的节省。在这个例子中，地址线 $\mathbf{a_{0}}$ 与 $\mathbf{a_{1}}$ 通过行地址选择（row address selection，$\overline{\text{RAS}}$）[^9]解复用器，选择一整行存储单元的地址线。读取时，所有这些存储单元的内容都会提供给列地址选择（column address selection，$\overline{\text{CAS}}$）多路复用器（multiplexer）。根据地址线 $\mathbf{a_{2}}$ 与 $\mathbf{a_{3}}$，其中一列的内容随后被提供给 DRAM 芯片的数据引脚（pin）。这会在多个 DRAM 芯片上并行发生多次，以产生与数据总线宽度对应的总位数。

对于写入操作，新的存储单元值会被放到数据总线上；当存储单元通过 $\overline{\text{RAS}}$ 与 $\overline{\text{CAS}}$ 被选中时，这个值就会被存入该单元。这是相当直观的设计。不过在现实中，显然还有更多复杂之处。必须规定信号发出之后，读取数据出现在数据总线之前会有多长延迟。如上一节所述，电容无法瞬时放电。来自存储单元的信号非常微弱，必须被放大（amplify）。对写入操作来说，也必须规定 $\overline{\text{RAS}}$ 与 $\overline{\text{CAS}}$ 完成之后，数据必须在总线上保持多久，才能成功地把新值存入存储单元（再次强调，电容不会瞬时充电或放电）。这些时序常数（timing constant）对 DRAM 芯片性能至关重要。下一节会讨论这些内容。

另一个次要的可扩展性问题是，把 30 条地址线连接到每个 RAM 芯片也并不可行。芯片引脚是宝贵资源。数据必须尽可能多地并行传输（例如以 64 位为一批），这已经够“糟”了。内存控制器还必须能够寻址每个 RAM 模块（module，即 RAM 芯片的集合）。如果出于性能原因需要并行访问多个 RAM 模块，而每个 RAM 模块都需要自己的一组 30 条或更多地址线，那么对于 8 个 RAM 模块来说，内存控制器仅地址处理就需要多达 240 多个引脚。

为了解决这些次要的可扩展性问题，DRAM 芯片长期以来都会自行对地址进行多路复用。这意味着地址会分两部分传输。第一部分由地址位（图 2.7 示例中的 $\mathbf{a_{0}}$ 与 $\mathbf{a_{1}}$）组成，用于选择行。这个选择会保持有效，直到被撤销。然后第二部分，即地址位 $\mathbf{a_{2}}$ 与 $\mathbf{a_{3}}$，用于选择列。关键差异在于，只需要两条外部地址线。[^译注]还需要少量额外线路来指示 $\overline{\text{RAS}}$ 与 $\overline{\text{CAS}}$ 信号何时可用，但与地址线数量减半相比，这只是很小的代价。不过，这种地址多路复用也会带来它自身的问题。我们将在 2.2 节讨论这些问题。

[^6]: 我讨厌这些 [SI 前缀（prefix）](https://www.nist.gov/pml/weights-and-measures/metric-si-prefixes)。对我来说，1 giga-bit 永远是 2<sup>30</sup>，而不是 10<sup>9</sup> bit。

[^7]: 现代 DRAM 类型（如 DDR3）可以自动调整时序，但可以容忍的范围有限。

[^8]: 多路复用器与解复用器是等价的，而且这里的多路复用器在写入时需要像解复用器一样工作。所以从现在开始我们会忽略二者差异。

[^9]: 名字上的线表示信号是反相的（negated）。

[^译注]: 这里的意思单看字面可能比较难理解。以图 2.7 为例，内存由 4 × 4 个存储单元组成，对外仅提供两条地址线。选择存储单元时，首先由两条外部地址线指定行，并将结果暂存起来；接着再利用同样的两条外部地址线指定列。这样便可以只用两条外部地址线来指定 16 个存储单元中的一个。详见 2.2 节。
